Technical Communication

Snake Cube Puzzle
3×3 Manual

Designed a comprehensive, user-friendly visual manual for solving the 3×3 snake cube puzzle. The project applied technical communication principles — using clear diagrammatic step-by-step instructions accessible to a general, non-technical audience.

Technical Writing Visual Design Documentation
Step-by-step visual guide General audience focused 3D spatial reasoning

The Snake Cube Puzzle is a spatial reasoning puzzle consisting of 27 wooden blocks connected by an elastic cord, which must be folded into a 3×3×3 cube. While the solution exists, communicating it clearly to a first-time user is a genuine design challenge — the puzzle operates in three dimensions, and any ambiguity in the instructions leads directly to frustration.

This project was an exercise in technical communication principles: understanding your audience, choosing the right visual representation, and sequencing information in a way that builds understanding progressively. The same skills that make a good circuit schematic or datasheet make a good user manual.

The manual uses a consistent visual vocabulary — isometric diagrams of the block positions, colour-coded fold direction indicators, and numbered step callouts — that allows a reader to follow the solution without needing to read extended prose. Each step shows exactly one action, and the subsequent step image confirms the expected result.

Technical communication is a core engineering skill. Writing the ETL CAN Hitch datasheet, documenting OptiBOM's API integration, and designing this manual all drew on the same discipline: making complex information accessible to the people who need it.

Key Principles Applied

  • Audience Analysis: Designed for non-technical users with no prior puzzle experience — no assumed spatial reasoning background
  • Visual Hierarchy: Large isometric diagrams as primary information carriers; text as secondary annotation only
  • Progressive Disclosure: Each step reveals exactly one new action, building complexity gradually
  • Consistency: Uniform visual style, colour coding, and annotation conventions maintained throughout the manual
