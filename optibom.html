OptiBOM
Automation

Architected a hybrid procurement automation tool at Electrans Technology using Node.js and Google Apps Script, integrating Digi-Key and Mouser APIs to automate BOM stock validation and EOL detection. Reduced manual analysis time by 90%+.

Node.js Google Apps Script REST API OAuth 2.0 JavaScript
90%+ manual time eliminated Digi-Key & Mouser APIs integrated EOL detection automated

During my co-op at Electrans Technology, engineers were spending significant time manually cross-referencing Bills of Materials against distributor websites to check component availability and lifecycle status. OptiBOM was built to eliminate this entirely.

The system is architected as a hybrid tool — a Node.js backend handles the heavy API integration (authentication flows, rate limiting, response parsing), while a Google Apps Script frontend runs directly inside Google Sheets, the tool engineers already use for BOM management. This means zero friction adoption: engineers hit a button, the BOM validates itself.

Integrating the Digi-Key and Mouser REST APIs required implementing OAuth 2.0 authentication flows, handling paginated responses, and normalizing data from two different API schemas into a unified format. The system checks real-time stock levels, flags components approaching end-of-life, and surfaces pricing data — all in one pass.

The result was a 90%+ reduction in manual BOM analysis time. What previously took an engineer 30–60 minutes per BOM revision now runs in under 3 minutes automatically, with no possibility of human transcription errors.

Key Technologies

  • Node.js: Backend API integration layer — OAuth 2.0 token management, request batching, error handling
  • Google Apps Script: Frontend integration into Google Sheets; custom menu, UI prompts, cell-level data writing
  • Digi-Key API v3: Real-time stock validation, pricing, and lifecycle queries via authenticated REST calls
  • Mouser API: Secondary distributor integration for availability cross-referencing and redundancy
  • OAuth 2.0: Secure token-based authentication flow for both distributor APIs
